网站无法访问排查步骤
====================

如果输入 http://localhost:3000 无法访问，请按以下步骤排查：

1. 确认开发服务器是否正在运行
   - 查看命令行窗口，应该看到类似这样的信息：
     VITE v5.x.x  ready in xxx ms
     ➜  Local:   http://localhost:3000/
     ➜  Network: use --host to expose
   
   - 如果看不到这些信息，说明服务器没有启动，需要运行：
     npm run dev

2. 检查端口号
   - 如果3000端口被占用，Vite会自动使用其他端口（如5173）
   - 查看命令行输出中的实际地址，使用那个地址访问
   - 例如：如果显示 http://localhost:5173，就访问这个地址

3. 确认服务器正在运行
   - 命令行窗口必须保持打开状态
   - 如果关闭了命令行窗口，服务器就会停止
   - 需要重新运行 npm run dev

4. 检查防火墙
   - Windows防火墙可能会阻止本地服务器
   - 如果提示，选择"允许访问"

5. 尝试其他端口
   - 如果3000端口有问题，可以修改 vite.config.js：
     server: {
       port: 5173,  // 改为其他端口
       open: true
     }

6. 检查浏览器
   - 尝试使用不同的浏览器（Chrome、Firefox、Edge）
   - 清除浏览器缓存
   - 尝试无痕/隐私模式

7. 查看错误信息
   - 按 F12 打开浏览器开发者工具
   - 查看 Console 标签页是否有错误信息
   - 查看 Network 标签页，看请求是否成功

8. 重新安装依赖
   如果以上都不行，尝试：
   - 删除 node_modules 文件夹
   - 删除 package-lock.json（如果有）
   - 重新运行：npm install
   - 然后运行：npm run dev

常见问题：
- 端口被占用：Vite会自动使用其他端口，查看命令行输出
- 服务器未启动：必须运行 npm run dev 才能访问
- 命令行窗口关闭：关闭后服务器停止，需要重新运行
