# 网站部署说明

## 为什么只能通过 npm run dev 访问？

`npm run dev` 启动的是**本地开发服务器**，它只在您的电脑上运行，只能通过 `http://localhost:3000` 访问。这是正常的开发模式。

要让其他人通过网址访问您的网站，需要将网站**部署到互联网服务器**上。

## 部署方法

### 方法一：使用 Vercel（推荐，最简单）

1. **安装 Vercel CLI**（如果还没有）：
   ```bash
   npm install -g vercel
   ```

2. **在项目目录下运行**：
   ```bash
   vercel
   ```

3. 按照提示操作，Vercel 会自动：
   - 构建您的网站
   - 部署到互联网
   - 提供一个免费的网址（如：`your-site.vercel.app`）

### 方法二：使用 Netlify

1. **安装 Netlify CLI**：
   ```bash
   npm install -g netlify-cli
   ```

2. **构建项目**：
   ```bash
   npm run build
   ```

3. **部署**：
   ```bash
   netlify deploy --prod
   ```

### 方法三：使用 GitHub Pages

1. **构建项目**：
   ```bash
   npm run build
   ```

2. 将 `dist` 文件夹的内容上传到 GitHub Pages

### 方法四：部署到自己的服务器

1. **构建生产版本**：
   ```bash
   npm run build
   ```

2. 构建完成后，会在项目根目录生成 `dist` 文件夹

3. 将 `dist` 文件夹的内容上传到您的服务器（如：Nginx、Apache 等）

## 构建生产版本

无论使用哪种部署方法，都需要先构建生产版本：

```bash
npm run build
```

构建完成后，会在项目根目录生成 `dist` 文件夹，这个文件夹包含了可以部署到服务器的所有文件。

## 本地预览生产版本

构建完成后，可以在本地预览生产版本：

```bash
npm run preview
```

这会启动一个本地服务器，预览构建后的网站效果。

## 注意事项

1. **开发模式**（`npm run dev`）：用于开发和测试，只能本地访问
2. **生产模式**（`npm run build` + 部署）：用于正式发布，可以通过网址访问
3. 部署后，网站会有一个公开的网址，任何人都可以访问

## 推荐部署平台

- **Vercel**：最简单，自动部署，免费
- **Netlify**：功能强大，免费
- **GitHub Pages**：适合开源项目，免费
- **Cloudflare Pages**：速度快，免费

选择其中一个平台，按照其文档进行部署即可。
